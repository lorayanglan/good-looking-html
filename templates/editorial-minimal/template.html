Ghost What Changed · The Five Shifts
The New Engineering Stack
From Writing Code
to Directing Agents
01
The bottleneck is no longer typing speed
Karpathy compares today's token quota to a PhD student's GPU time. The anxiety isn't "am I coding fast enough" — it's "am I running enough agents in parallel."
02
The unit of work is now a feature, not a function
Agents receive macro goals — "refactor this module," "research this problem" — and run for 20-minute stretches. Human review happens at the end, not the middle.
03
Agents need identity, memory, and a single inbox
The next layer isn't better models — it's persistent agents with consistent character and a unified interface (e.g., WhatsApp) replacing six fragmented apps.
Key Metric

The real skill is no longer writing code — it's knowing how to parallelise agent tasks without creating conflicts between concurrent runs.

AutoResearch

Karpathy's goal: minimal human tokens in, massive autonomous output out. An agent reads a field's literature, runs experiments, synthesises findings — unattended.
