🎯 The Next Bottleneck Deciding
What
When build speed accelerates, the bottleneck shifts to deciding what to build. Point agents at feedback streams, bug logs, and user signals to surface the next move.
"Scarcity moves from building to deciding."
🔭 Product Design Principle Forward
6 mo
Design for the model 6 months ahead. Give fewer constraints, more tools. If today's model is 80% there — that's a product opportunity in half a year.
"Model latent demand becomes product latent demand."
🛠️ The Role Shift Builder
Builder
PM, engineer, designer, data scientist — roles are blurring. The converging archetype: someone who defines goals, judges outputs, understands users, and ships.
"Software Engineer as a title may be fading."
Reusable Principles · From the Source
How to Work Like an Agent
Six principles from Boris Cherny you can start applying today.
🗺️
Plan First, Execute Second
Use plan mode for complex tasks. Let the agent read context and propose steps. Align before execution — especially on high-stakes work.
📡
Point Agents at Input Streams
Feedback, logs, knowledge base, chat history. Agents can surface what to do next. Don't just give agents tasks — let them find the tasks.
⚡
Run Agents in Parallel
Treat agents like background threads. Start multiple, let them work, review outputs. Speed is the competitive advantage — compress clock time.
🔒
Keep Human Checkpoints
Production code, external sends, irreversible actions — require human review. Agent velocity doesn't mean skipping accountability gates.
🔋
Use Strong Models Early
Cheap models needing more correction may cost more in total. Prove value first with strong models and enough tokens — optimize cost after.
🔭
Design for Future Models
If today's model is almost there, 6 months later it's a product. Build the scaffolding now. Latent capability is latent opportunity.
